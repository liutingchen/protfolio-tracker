# Portfolio Tracker 📈

记录股票组合里的每一笔交易和交易原因，并据此自动生成一张 **MarketSurge 风格的组合走势图**（周 K 线 + 成交量 + **10 周均线**）。每笔买卖会作为标记打在走势图上，**鼠标悬停即可看到当时的交易原因**。

本地运行、数据存在本机、股价自动抓取，无需注册任何账号。

---

## 快速开始

依赖已经装好（在 `.venv` 里）。直接运行：

```bash
cd ~/portfolio-tracker
./run.sh
# 或者： .venv/bin/python app.py
```

然后浏览器打开 **http://127.0.0.1:5174**

> 首次打开会看到一组示例数据（NVDA / AAPL / MSFT / AMZN）。点交易记录右上角的 **「清空全部」** 即可清掉，开始录入你自己的交易。

如果环境需要重建：

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## 功能

- **多用户账号**：打开后先**注册 / 登录**（邮箱 + 密码）。每个账号的组合与交易**完全隔离**，只能看到自己的数据。密码用 `werkzeug` 加盐哈希存储；登录态用签名 Cookie（`HttpOnly` + `SameSite=Lax`），重启服务不掉登录。想直接体验可用内置示例账号 **`demo@demo.com` / `demo1234`**（建议自己注册后删掉它）。
- **注册邮箱验证 + 忘记密码找回**：注册后会收到**验证邮件**，点链接验证后才能登录（未验证登录会被拦，并可一键「重新发送」）；登录页有「忘记密码」，填邮箱收到**重置链接**即可设新密码。Token 随机、存哈希、一次性、带过期（验证 24h / 重置 1h）；忘记密码不泄露邮箱是否注册。**需配置 SMTP**（见下方「邮件配置」），否则邮件只打印到服务器日志（开发模式）。
- **多组合管理**：右上角可**新建 / 切换 / 重命名 / 删除**多个组合（如「主仓」「波段仓」「实验仓」）。每个组合**完全独立**——各自的交易记录、起始本金和视图模式互不影响；下拉框里会显示每个组合的交易笔数。
- **全部组合合并总览**：下拉框选 **「▦ 全部组合（合并）」**，把所有组合并成**一条总资产走势图**（含 10 周均线）、汇总统计（总市值/总盈亏/总收益率）和**按代码合并的持仓**（跨组合的同一只股票自动加总、成本取加权均价）。合并视图为**只读**——增删改请回到具体组合。
- **导入当前持仓**：点「添加交易」卡片右上角的 **⇪ 导入持仓**，一次性录入你**现在的现金**和**已持有的仓位**（代码 / 股数 / 平均成本）。按「截至日期」为每个持仓生成一笔建仓买入，并把现金 + 持仓成本计入账户本金——导入后现金正好等于你填的金额，盈亏用实时价计算。免去逐笔补录历史交易，可重复使用（追加资金/仓位）。
- **修改现金**：点顶部统计区的 **「现金」** 卡片（鼠标悬停显示 ✎），输入目标金额即可。后端会自动调整起始本金使现金等于该值，**持仓、盈亏、走势图都不受影响**——适合存取现金、对账时直接把现金校准到券商显示的数。
- **个股 K 线图（周/日切换）**：在「当前持仓」表里**点击股票代码**，弹出该股票的 K 线图 + 成交量柱，并标出你的买卖点。右上角可切 **周线 / 日线**：
  - **周线**：10 周均线 + 40 周均线（MarketSurge 经典趋势线）
  - **日线**：EMA(10)、EMA(21)、SMA(50)、SMA(150)、SMA(200)（完整复刻 MarketSurge 日线均线）
  - 十字光标悬停显示数据窗：日期、开/高/低/收、涨跌幅、Cls Range、成交量、Vol%（当日量 vs 50 日均量），以及当前周期每条均线的值和现价距均线的百分比。
- **录入交易**：日期、代码、买/卖、股数、价格、手续费、**交易原因/笔记**。可随时编辑或删除。
- **组合走势图**（用 TradingView Lightweight Charts 绘制，和 MarketSurge 观感一致）：
  - **周线**：把每天的组合总价值聚合成周 K 线（开/高/低/收）。
  - **日线**：每日组合价值的面积曲线。
  - **10 周均线**（蓝线）—— 你要求的核心指标；周线下满 10 周后开始绘制。
  - **40 周均线**（橙线，可选）—— 欧奈尔体系里的长期趋势线（≈200 日线）。
  - **成交量柱**：用你自己每周的买卖金额表示，净买入为绿、净卖出为红。
  - **交易标记**：买=绿色上箭头，卖=红色下箭头，标签是代码；**悬停显示股数、价格和交易原因**。
- **统计区**：账户总值、累计盈亏、收益率、已实现/未实现盈亏、持仓市值、现金、持仓数。
- **持仓表**：每只股票的股数、均价（移动平均成本法）、现价、市值、浮动盈亏。

## 走势图的三种纵轴（右上角切换）

| 模式 | 含义 | 是否需要起始本金 |
|------|------|------------------|
| **账户价值** | 起始本金 + 累计盈亏，即账户净值随时间变化（最像「股价」） | 需要设置起始本金 |
| **累计盈亏** | 已实现 + 未实现盈亏，从 0 出发的资金曲线 | 不需要 |
| **指数 (=100)** | 把账户价值归一化到 100 起步，看相对涨幅（最像 MarketSurge 的相对价格） | 需要起始本金 |

**起始本金**：在工具栏里填你投入的总本金后点「保存」。仅凭交易流水无法推断你存入了多少钱，所以：
- 想看「账户净值曲线」或「收益率」——填上起始本金；
- 只想看「赚/亏了多少」——留 0，用「累计盈亏」模式即可。

## 价格数据来源

组合价值用**真实历史收盘价**（复权）计算，按优先级自动尝试：

1. **stockanalysis.com**（默认，免费无需 key，复权日线）
2. **yfinance / Yahoo**（免费无需 key）
3. **Stooq**（可选，设环境变量 `STOOQ_API_KEY`）
4. **Alpha Vantage**（可选，设环境变量 `ALPHAVANTAGE_API_KEY`，免费 key：alphavantage.co）

抓到的价格会缓存在本地数据库，重复打开很快。点工具栏 **「↻ 刷新股价」** 可重新抓取最新价。

若某个源临时被限流，会自动切下一个；都失败时图表仍会显示（仅按现金流），并在顶部给出提示。此时设置 `ALPHAVANTAGE_API_KEY` 是最稳的兜底：

```bash
ALPHAVANTAGE_API_KEY=你的key ./run.sh
```

## 部署上线（可选）

本地自用直接 `./run.sh`。要放到公网，有两种方式：

### 方式一：Railway（推荐）

仓库已带 `Dockerfile`（锁定 Python 3.12），Railway 会直接用它构建。用 CLI 从本地部署：

```bash
npm i -g @railway/cli      # 安装 CLI
railway login              # 浏览器登录（只有这步需你本人操作）
railway init               # 新建项目
railway up                 # 用 Dockerfile 构建 + 部署
railway domain             # 生成公网 HTTPS 域名
```

部署后在 Railway 面板给该服务加一个 **Volume**，挂载路径填 **`/data`** —— 数据库和会话密钥存这儿，永久保存。`PORTFOLIO_ENV=production`、`DATA_DIR=/data` 已写进 Dockerfile 无需手配；`SECRET_KEY` 会自动生成并存到 Volume。

> ⚠️ 一定要加 `/data` 这个 Volume，否则每次重新部署数据会重置。

### 方式二：自己的服务器 / 其它平台

用 gunicorn 启动（生产 WSGI，多 worker）：

```bash
./run_prod.sh
# 等价于： PORTFOLIO_ENV=production gunicorn --preload -w 2 -k gthread --threads 4 -b 127.0.0.1:5174 app:app
```

`PORTFOLIO_ENV=production` 会开启 **HTTPS-only Cookie**（`Secure`）并信任反向代理头（ProxyFix）。

2）**前面挂反向代理做 HTTPS**。推荐 Caddy（自动签发证书）：

```
your.domain.com {
    reverse_proxy 127.0.0.1:5174
}
```

（nginx 同理，记得转发 `X-Forwarded-Proto`、`X-Forwarded-For`。）

**已内置的安全措施**：
- 密码 `werkzeug` 加盐哈希；会话签名 Cookie（`HttpOnly` + `SameSite=Lax`，生产再加 `Secure`）。
- **CSRF**：双提交 Token（`csrftoken` Cookie + `X-CSRF-Token` 请求头），所有写接口校验。
- **登录限流**：同一 IP 5 分钟内最多 15 次登录/注册尝试。
- 跨用户数据隔离（越权返回 404）。

**按需再加固**：限流是单进程内存级（多 worker 各自计数，要强限流上 `flask-limiter` + Redis）；邮箱验证 / 找回密码 / 审计日志可按需补充；`data/secret_key`、`data/portfolio.db` 注意备份与权限、别提交版本库。

## 邮件配置（仅用于「忘记密码」）

重置密码邮件支持两种发送方式，按优先级：**① Resend（HTTPS API，推荐）→ ② SMTP**。都不配则进入**开发模式**：邮件内容（含链接）只打印到日志（本地看终端、线上 `railway logs` 找 `[DEV EMAIL …]`）。

> ⚠️ **Railway / Render / Fly 等平台默认封禁出站 SMTP 端口（25/465/587）**，所以在这些平台上 **SMTP 发不出去**（日志会报 `[Errno 101] Network is unreachable`）。在 Railway 上请用 **Resend**，它走 HTTPS(443) 不受影响。

### 方式一：Resend（推荐，适用于 Railway）

1. 注册 https://resend.com → API Keys → 创建一个 `re_...` key。
2. 配置环境变量：

| 变量 | 说明 |
|------|------|
| `RESEND_API_KEY` | Resend 的 `re_...` key |
| `EMAIL_FROM` | 发件地址。未验证域名时用 `onboarding@resend.dev`（只能发到你的 Resend 账户邮箱）；要给任意用户发信需在 Resend 验证自己的域名后用 `noreply@你的域名` |
| `APP_BASE_URL` | 邮件里链接的域名（线上已设为 Railway 域名）|

```bash
railway variables --set "RESEND_API_KEY=re_xxx" --set "EMAIL_FROM=onboarding@resend.dev"
```

### 方式二：SMTP（仅限允许出站 SMTP 的环境，如自己的 VPS）

| 变量 | 说明 |
|------|------|
| `SMTP_HOST` / `SMTP_PORT` / `SMTP_USER` / `SMTP_PASSWORD` | 标准 SMTP 参数（587 STARTTLS / 465 SSL）|
| `EMAIL_FROM` | 发件地址（默认 = `SMTP_USER`）|

## 说明 / 局限

- 组合价值用**复权收盘价**估值；你录入的成交价仅用于成本/盈亏统计，因此分红/拆股附近的估值与盈亏可能有微小差异，属正常。
- 没有价格的代码（拼写错误、退市、非美股）会在顶部提示，其市值按 0 计。
- 目前面向美股；其它市场可后续扩展数据源。
- 本地开发服务器，仅监听 127.0.0.1，数据不出本机。
- 账号体系面向**本地 / 小范围自用**：已做密码哈希、签名 Cookie 会话、跨用户数据隔离；尚未做邮箱验证 / 找回密码 / 登录限流 / CSRF token（同源 + `SameSite=Lax` Cookie 已覆盖常见情况）。若要公网部署，建议再加 HTTPS、正式 WSGI（如 gunicorn）、限流与 CSRF 防护。会话密钥存于 `data/secret_key`（请勿提交到版本库）。
